1. 과학적 목표와 검증 질문

2000-2020
Reproduction

공식 기간에서 같은 입력자료와 전처리를 적용했을 때 SM 재현성 평가

2021+
Extension

공식 산출물 이후 기간까지 자연스럽게 이어지는 GSSM-compatible extension 준비

LST_Diff
Key diagnostic

일평균 LST와 day-night contrast가 SM 오차에 주는 영향 분리

핵심 질문

  • 공식 GSSM과의 차이가 모델 학습 때문인지, 입력 predictor 전처리 때문인지 분리
  • NDVI/EVI smoothing, LST gap filling, day/night LST 조합 중 주요 오차 요인 확인
  • 2021년 이후 확장 시 공식 2000-2020 산출물과 불연속이 작은 입력자료 구성 탐색
LST_Diff

낮 LST에서 밤 LST를 뺀 값. 주야간 온도 차이와 열적 건조도 정보

Gap filling

구름이나 품질 문제로 빠진 pixel 값을 시간 패턴이나 보조자료로 채우는 과정

Blending

raw 관측값과 gap-filled 예측값을 합치는 과정. 관측값 우선 사용

현재 결론: 가장 큰 과학적 병목은 RF 모델 구조보다 gap-filled/blended LST, 특히 LST_Diff 재현

3. 전처리 workflow

Raw inputMODIS vegetation, MODIS LST, ERA5, SoilGrids, 공식 GSSM sample 입력
VegetationNDVI/EVI Savitzky-Golay smoothing 후 daily interpolation
LSTday/night LST 별도 처리 후 LST_DAILY와 LST_Diff 계산
Sampling공식 GSSM validation sample 위치와 날짜에서 predictor 비교
Statisticsn, Bias, MAE, RMSE, r, R²로 predictor와 SM 오차 정량화

분석에서 고정한 파생 변수

LST_DAILY = (LST_Day + LST_Night) / 2 LST_Diff = LST_Day - LST_Night
Savitzky-Golay smoothing

시계열 형태는 유지하면서 짧은 기간의 튐과 관측 노이즈를 줄이는 polynomial smoothing

Daily interpolation

16일 간격 MODIS 식생값 사이를 보간해 매일 값이 있는 predictor로 변환

Bias / MAE / RMSE / r / R²

Bias는 평균 방향, MAE/RMSE는 오차 크기, r/R²는 선형 유사성

LST_DAILY: 평균 열 상태. LST_Diff: 주야간 온도 차이와 열적 건조도. 두 변수의 분리 검증 필요

4. 검증 설계

SM 재현 실험

  • 대상 기간: 2010년 1월과 7월
  • 학습 표본: RF training sample 최대 50,000개
  • 검증 표본: 공식 GSSM validation sample, 최대 50개/day
  • 비교 대상: 공식 GSSM SM과 직접 생성 SM

Predictor 오차 분해

  • 같은 날짜, 같은 sample 위치의 predictor 값 직접 비교
  • LST_DAILY, LST_Diff, NDVI_SG, EVI_SG 개별 평가
  • 통계지표: 표본 수 n, Bias, MAE, RMSE, Pearson r, R²
  • raw-valid pixel과 raw-gap pixel 분리로 LST gap filling 효과 확인
해석 기준: r이 높더라도 RMSE가 크면 물리량 크기 차이가 SM 산출에 영향

7. 식생 전처리 검토

원본 방법

  • 원본 GSSM: MOD13A2 NDVI/EVI 사용
  • 16일 composite에 Savitzky-Golay smoothing 적용
  • smoothing된 시계열을 daily scale로 interpolation
  • OEEL SavatskyGolayFilter 원본 알고리즘 확인

분석 결과

  • OEEL original SG 구현 완료
  • 2010년 1월 SM RMSE는 소폭 증가
  • 2010년 7월 개선은 매우 작음
  • 현재 오차 구조를 식생 전처리만으로 설명하기 어려움

해석: NDVI/EVI는 중요 predictor지만 현재 재현성 차이의 지배 변수는 아님. 이후 검증은 LST 전처리에 집중

8. 원본 LST 자료의 성격

  • day LST와 night LST를 따로 읽은 뒤 LST_DAILY와 LST_Diff 계산

TFA

Typical annual temperature cycle. 여러 해 자료로 만든 날짜별 평균 LST 계절 곡선

CFSv2 anomaly

현재 재분석 LST가 평균 상태에서 벗어난 양. TFA에 더해 날씨 변동 반영

raw-valid

MODIS 원 관측값이 실제로 존재하는 pixel. 여기서는 저자 LST와 거의 일치

해석: raw MODIS가 존재하는 pixel에서는 저자 LST와 거의 동일. 오차는 결측을 채우는 gap-fill 영역에서 발생

11. 원본 LST 전처리 재현 전략

확인된 원본 접근

  • 장기 평균 계절성인 MODIS TFA 선계산
  • CFSv2 기반 anomaly로 결측 LST 보완
  • 예측 LST와 raw MODIS LST blend
  • day와 night 별도 처리 후 LST_DAILY와 LST_Diff 계산

현재 제약

  • Europe 전체 TFA asset export가 GEE storage quota 250 GiB에 막힘
  • 작은 ROI 또는 기간 단위 TFA 생성 후 검증 필요
  • 검증 기준: 저자 LST와의 Day, Night, Daily, Diff 통계 일치도
  • 2021+ 확장: MODIS C6.1을 C6 baseline에 harmonization한 뒤 적용
Harmonization

MODIS C6.1 자료를 공식 GSSM의 C6 baseline과 통계적으로 이어지도록 보정하는 과정

ROI

Region of interest. 먼저 작은 지역에서 검증한 뒤 Europe 전체로 확장

Thermal contrast

낮과 밤의 지표면온도 차이. LST_Diff가 이 정보를 대표

핵심: 단순히 "LST 값을 채우는 것"이 아니라, 공식 GSSM이 학습한 thermal contrast 분포 재현
